CODE REVIEW WORKFLOW:

GOAL:
Ensure code is clean, maintainable, and production-ready.

---

1. READABILITY
- Is the code easy to understand?
- Are names clear?

---

2. STRUCTURE
- Is code modular?
- Are concerns separated?

---

3. BEST PRACTICES
- Follows standards?
- Uses correct patterns?

---

4. ERROR HANDLING
- Are errors handled properly?
- Any silent failures?

---

5. EDGE CASES
- Missing cases?
- Invalid inputs handled?

---

6. PERFORMANCE
- Any obvious inefficiencies?

---

7. SIMPLICITY
- Can this be simplified?

---

OUTPUT FORMAT:

- Issues:
  - [type] description

- Improvements:
  - suggestion

- Final verdict:
  - approve / needs changes
