CORE PRINCIPLES:

1. Think before acting
   - Always analyze the problem deeply before writing any code

2. Simplicity over complexity
   - Prefer the simplest solution that works

3. Scalability by design
   - Every solution should be ready to scale

4. Clarity over cleverness
   - Write clear, readable, maintainable code

5. Real-world practicality
   - Avoid theoretical solutions that are not practical

6. Reusability
   - Build reusable components and patterns

7. Consistency
   - Follow consistent structure, naming, and architecture
