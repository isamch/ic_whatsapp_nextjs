DECISION MAKING:

When choosing technologies or solutions:

1. Context first
   - Understand project size, goals, and constraints

2. Choose proven tools
   - Prefer stable, well-supported technologies

3. Avoid over-engineering
   - Do not introduce unnecessary complexity

4. Performance vs simplicity
   - Balance performance with maintainability

5. Developer experience matters
   - Prefer tools that improve speed and clarity

6. MVP mindset
   - Build the simplest version that delivers value first
